# `http://127.0.0.1:8080/` 首页工作台演示手册

这份文档**只针对首页工作台**来写，不要求你切去 `docs` 或 `8081` 页面。

目标是：

> 只通过 `http://127.0.0.1:8080/` 这个聊天式工作台，逐步演示 ESM3 的核心能力，并让组里的人直观看到它“能干什么”。

---

## 1. 首页里能展示什么

在当前首页工作台中，你可以直接演示或间接展示以下 7 项能力：

1. `Sequence generation`（序列生成）
2. `Structure prediction`（结构预测）
3. `Inverse folding`（逆折叠）
4. `Function conditioned generation`（功能条件化生成）
5. `Multi-modal reasoning`（多模态推理）
6. `Iterative design`（迭代设计）
7. `Evolution simulation`（进化模拟）

其中：

- `逆折叠` 和 `功能条件化生成` 有**独立模式卡片**
- `结构预测` 会在 `迭代设计` 结果里通过 `完整 JSON` 和 `history / structure_data` 被展示出来
- `多模态推理`、`迭代设计`、`进化模拟` 都在 `迭代设计` 模式里演示

---

## 2. 会前检查

你打开 `http://127.0.0.1:8080/` 后，先确认：

- 左侧 `Agent API` 状态为正常
- 左侧 `ESM3 Runtime` 状态为正常

如果这两项都正常，就可以直接开始演示。

---

## 3. 推荐演示顺序

如果你想让听众最好理解，建议按这个顺序来：

1. 先展示 `迭代设计`，让大家理解“这不是普通聊天框，而是一个蛋白设计工作台”
2. 再展示 `逆折叠`
3. 再展示 `功能条件化生成`
4. 最后回到 `迭代设计`，用高级输入展示：
   - 多模态推理
   - 结构参与设计
   - 进化模拟参数

---

## 4. 你在首页可以直接做的 6 组典型演示

下面所有操作都只在：

```text
http://127.0.0.1:8080/
```

完成。

---

## 演示 A：最基础的迭代设计

### 目标

先让大家看到：这个系统会自动调用 ESM3 生成候选、评估候选、返回最佳序列。

### 页面设置

- 模式：`迭代设计`
- 最大迭代轮数：`3`
- 每轮候选数：`4`
- 提前停止容忍轮数：`2`
- 种群大小：`8`
- 精英保留数：`2`
- 父代池大小：`4`
- 每个父代突变位点数：`3`

### 输入框里填

```text
请自动设计 GFP 并迭代优化
```

### 演示时你要指给大家看

- 中间聊天区：会有任务总结
- 右侧 `最新结果摘要`
- 右侧 `最佳序列`
- 右侧 `候选历史`
- 右侧 `完整 JSON`

### 这一步主要说明什么

- 这不是单轮序列生成，而是自动化设计流程
- ESM3 的生成能力已经被 Agent 编成了一个工作流

### 你可以这样讲

> “这一步演示的是最核心的能力：ESM3 生成候选，Agent 自动评估、筛选，再进行下一轮优化。”

---

## 演示 B：Sequence generation（序列生成）

### 目标

只强调“ESM3 能生成新序列”。

### 页面设置

- 模式：`功能条件化生成`
- 基础序列：留空
- 目标长度：`128`
- 候选数：`2`
- 温度：`0.8`
- `num_steps`：`8`
- 功能关键词：

```text
fluorescent protein
```

### 输入框里填

```text
请生成两个候选序列
```

### 演示时你要指给大家看

- 右侧 `最佳序列`
- 右侧 `候选历史`
- `完整 JSON` 中的 `sequences`

### 这一步主要说明什么

- ESM3 可以直接生成候选蛋白序列
- 不是只能补全输入，而是可以做条件化生成

### 建议话术

> “这里最核心的是 sequence generation：给定一定条件后，ESM3 能直接产出新的候选序列。”

---

## 演示 C：Structure prediction（结构预测）

### 目标

说明 ESM3 不只是会生成序列，还会在结构轨道上参与评估。

### 页面设置

沿用 **演示 A** 的设置即可。

### 输入框里填

```text
请自动设计 GFP 并迭代优化，重点展示结构评估结果
```

### 演示时你要指给大家看

在右侧 `完整 JSON` 中展开：

- `history`
- 每条记录里的 `structure_data`

你会看到类似：

- `structure`
- `confidence`

### 这一步主要说明什么

- ESM3 在设计流程中不仅负责出序列，也负责结构轨道上的结果支持
- 也就是说，这个系统内部已经把 structure prediction 纳入设计闭环

### 建议话术

> “这里虽然首页没有单独做一个‘结构预测按钮’，但在设计闭环里，ESM3 的结构预测已经真实参与了评估。”

---

## 演示 D：Inverse folding（逆折叠）

### 目标

说明 ESM3 可以 `structure -> sequence`。

### 页面设置

- 模式：`逆折叠`
- PDB 路径：

```text
/mnt/disk3/tio_nekton4/esm3/data/lutn.pdb
```

- 候选数：`2`
- 温度：`0.8`
- `num_steps`：`1`

### 输入框里填

```text
请根据这个结构生成两个候选序列
```

### 备选操作

如果你不想填路径，可以：

- 点击 `上传 PDB 文件`
- 或直接把 PDB 文本粘到 `PDB 文本` 输入框里

### 演示时你要指给大家看

- 右侧 `最佳序列`
- 右侧 `候选历史`
- `完整 JSON` 中的 `sequences`

### 这一步主要说明什么

- ESM3 不只是 sequence→structure，也能 structure→sequence
- 这正是逆折叠的意义

### 建议话术

> “这一步是 inverse folding：给结构，反推出 compatible sequence。这是蛋白设计里很关键的一类能力。”

---

## 演示 E：Function conditioned generation（功能条件化生成）

### 目标

说明 ESM3 能按功能目标来约束序列生成。

### 页面设置

- 模式：`功能条件化生成`
- 目标长度：`128`
- 候选数：`2`
- 温度：`0.8`
- `num_steps`：`8`
- 功能关键词：

```text
fluorescent protein
```

### 如果你想更正式一点

点击：

- `插入注释模板`

然后把生成的 JSON 保留为例如：

```json
[
  {
    "label": "fluorescent protein",
    "start": 1,
    "end": 128
  }
]
```

### 输入框里填

```text
请在功能约束下生成候选序列
```

### 演示时你要指给大家看

- 候选序列返回
- 功能约束已经进入输入区
- 右侧完整结果里保留了生成结果

### 这一步主要说明什么

- ESM3 的 function 轨道不是摆设
- 它可以参与序列生成约束

### 建议话术

> “这一步说明 ESM3 不只是按序列或结构条件生成，它还能按功能目标来生成候选序列。”

---

## 演示 F：Multi-modal reasoning（多模态推理）

### 目标

说明首页工作台里的 `迭代设计` 已经不只是文本任务，而是可以把 `text + sequence + structure + function` 一起送进同一个设计流程。

### 页面设置

- 模式：`迭代设计`

基础参数：

- 最大迭代轮数：`3`
- 每轮候选数：`4`
- 提前停止容忍轮数：`2`
- 种群大小：`8`
- 精英保留数：`2`
- 父代池大小：`4`
- 每个父代突变位点数：`3`

高级输入：

- 参考序列：

```text
MSKGEELFTGVV
```

- 结构路径：

```text
/mnt/disk3/tio_nekton4/esm3/data/lutn.pdb
```

- 功能关键词：

```text
fluorescent protein
```

- 目标长度：

```text
128
```

### 输入框里填

```text
请基于给定结构和功能约束继续优化 GFP
```

### 演示时你要指给大家看

右侧 `最新结果摘要` 中的：

- `输入模态`

如果系统正常，你应该会看到类似：

```text
sequence, structure, function
```

### 这一步主要说明什么

- 这是多模态推理，不是单独某个 API
- 文本任务、参考序列、结构信息、功能约束都进入了同一个设计流程

### 建议话术

> “这一页最大的价值就在这里：不仅可以聊天式下任务，还能把 sequence、structure、function 一起送进一个统一的设计流程。”

---

## 演示 G：Evolution simulation（进化模拟）

### 目标

展示这不是简单 top-k 迭代，而是带种群与代际统计的进化式优化。

### 页面设置

- 模式：`迭代设计`

参数改成：

- 最大迭代轮数：`4`
- 每轮候选数：`4`
- 提前停止容忍轮数：`3`
- 种群大小：`12`
- 精英保留数：`2`
- 父代池大小：`4`
- 每个父代突变位点数：`3`

### 输入框里填

```text
请自动设计 GFP，并用更偏进化搜索的方式迭代优化
```

### 演示时你要指给大家看

右侧新增的：

- `代际统计`

这里会列出每一代的：

- `best`
- `avg`
- `worst`
- `population`
- `elite`
- `parent_pool`
- `mutations`

### 这一步主要说明什么

- 当前系统里已经不是简单的“拿最佳候选做下一轮”
- 而是一个 population-based evolutionary search

### 建议话术

> “这里的 Agent 已经不仅是 workflow automation，而是一个带种群参数和代际统计的进化式优化框架。”

## 5. 七项能力和首页的对应关系

你可以直接用这张表在组会上总结：

| ESM3 功能 | 在首页怎么展示 |
|---|---|
| Sequence generation | `功能条件化生成` 模式直接出候选序列；`迭代设计` 模式也会生成候选 |
| Structure prediction | `迭代设计` 结果里的 `完整 JSON -> history -> structure_data` |
| Inverse folding | `逆折叠` 模式 |
| Function conditioned generation | `功能条件化生成` 模式 |
| Multi-modal reasoning | `迭代设计` 模式里的高级输入区 |
| Iterative design | `迭代设计` 模式 |
| Evolution simulation | `迭代设计` 模式 + 右侧 `代际统计` |

## 7. 会前彩排建议

在组会前至少按这个顺序点一遍：

1. `迭代设计`
2. `逆折叠`
3. `功能条件化生成`

- `最新结果摘要`
- `最佳序列`
- `与上一轮最佳序列的差异`
- `代际统计`
- `候选历史`
- `完整 JSON`

---

## 8. 如果现场有一步失败怎么救场

优先保留这三项：

1. `迭代设计`
2. `逆折叠`
3. `功能条件化生成`

如果多模态或进化统计现场表现不稳定：

- 直接把右侧 `完整 JSON` 打开
- 强调系统已经支持这些输入与返回结构
- 不要把组会拖进调试流程
